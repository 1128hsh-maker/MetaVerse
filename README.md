# 🎮 [프로젝트 제목] - 아케이드 미니게임 컬렉션

> Unity 엔진으로 개발된 2D 허브 시스템 및 다양한 아케이드 스타일 미니 게임을 포함하는 프로젝트.

## 🌟 프로젝트 개요

본 프로젝트는 두 가지 핵심 컴포넌트로 구성되어 있습니다.

1.  **메타버스/허브 씬:** 플레이어가 자유롭게 이동(`MetaVersePlayer.cs`)하며 게임을 선택하고, 최고 점수를 확인(`Score.cs`)하는 중앙 로비 역할을 합니다.
2.  **미니 게임 씬:** 허브에서 진입 가능한 **무한 타워 쌓기 게임(The Stack)** 및 **2D 플래피 버드 스타일 게임**을 포함합니다.

### 💡 주요 기술 및 특징

* **모듈형 UI 시스템:** `BaseUI.cs`, `UIManager.cs`를 기반으로 Home, Game, Score 등의 UI 상태를 체계적으로 전환하고 관리합니다.
* **전역 점수 관리:** `ScoreManager.cs` (DontDestroyOnLoad)를 통해 씬 이동 간에도 점수와 최고 기록(`BESTSCORE`, `BestCombo`)을 유지합니다.
* **씬 이동:** 트리거 영역(`ScenesMovement.cs`)을 통한 자연스러운 씬 전환을 구현합니다.

## 🕹️ 미니 게임 상세

### 1. 🏗️ 타워 쌓기 게임 (The Stack)

* **핵심 스크립트:** `TheStack.cs`
* **게임 로직:** 움직이는 블록을 정확한 타이밍에 멈춰서 타워를 쌓아 올립니다.
* **정교한 계산:** 이전 블록과의 오차(`ErrorMargin`)를 계산하여 블록을 자르고, 쌓기 카운트(`stackCount`)와 콤보(`comboCount`)를 증가시킵니다.
* **게임 오버:** 블록을 놓치는 경우 파괴 효과(`GameOverEffect()`)를 발생시키고 게임을 종료합니다.
* **점수 기록:** 최고 점수(`BestScore`)와 최고 콤보(`BestCombo`)를 `PlayerPrefs`를 사용하여 저장합니다.
* **폐기물 처리:** 화면 밖으로 떨어진 블록 파편(`Rubble`)은 `DestroyZone.cs`에 의해 정리됩니다.

### 2. 🐦 플래피 버드 클론

* **핵심 스크립트:** `Player.cs`, `Obstacle.cs`
* **게임 로직:** 마우스 클릭 또는 스페이스바 입력(`Player.cs`)을 통해 플레이어를 상승시키고, 자동으로 전진합니다.
* **무한 환경:** `BgLooper.cs`를 사용하여 배경과 장애물(`Obstacle`)을 효율적으로 재활용하고, 무한한 횡스크롤 환경을 만듭니다.
* **장애물:** `Obstacle.cs`는 랜덤한 위치와 크기의 통과 구멍을 생성하며, 통과 시 점수 획득 로직을 포함합니다.

## 💻 스크립트 구성 및 역할

| 파일 그룹 | 파일명 | 역할 및 기능 상세 |
| :--- | :--- | :--- |
| **타워 게임 (Stacker)** | `TheStack.cs` | 타워 블록 생성, 이동, 블록 커팅, 점수/콤보 계산 및 게임 오버/재시작 로직. |
| | `DestroyZone.cs` | 떨어진 블록 조각(Rubble)을 삭제하여 메모리 관리. |
| **플래피 버드** | `Player.cs`, `Obstacle.cs` | 플레이어 비행 물리, 충돌, 장애물 생성 및 재활용 로직. |
| | `BgLooper.cs` | 배경 및 장애물 순환 로직 (무한 횡스크롤 구현). |
| | `GameManager.cs` | 플래피 버드 씬의 게임 상태 및 점수 증가 관리 (구형 관리자). |
| **허브 & 이동** | `MetaVersePlayer.cs` | 키보드 입력(WSAD) 기반의 2D 플레이어 이동 및 `Anime.cs`를 이용한 애니메이션 제어. |
| | `ScenesMovement.cs` | 플레이어와의 충돌 시 특정 씬("TheStack")으로 이동. |
| | `PlayerCamera.cs` | 플레이어의 X, Y 위치를 추적하는 카메라 로직. |
| **전역 관리** | `ScoreManager.cs` | 씬 전환 시 파괴되지 않는 싱글톤으로 최고 점수 기록 저장 및 관리. |
| **UI 시스템** | `UIManager.cs` | UI 상태(`Home`, `Game`, `Score`) 전환 및 각 UI 컴포넌트 관리 (싱글톤). |
| | `BaseUI.cs` | 모든 UI 클래스의 추상 기본 클래스. 상태 기반 활성화/비활성화 로직 제공. |
| | `HomeUI.cs` | 게임 시작 및 종료 버튼을 포함하는 홈 화면 UI. |
| | `GameUI.cs` | 게임 중 현재 점수, 콤보, 최대 콤보를 표시하는 UI. |
| | `ScoreUI.cs` | 게임 종료 후 최종 점수, 콤보, 최고 기록을 표시하는 UI. |

## 🛠️ 설치 및 실행

### 1. 요구 사항

* Unity Editor (스크립트의 호환성을 위해 **Unity 2020 이상** 버전을 권장합니다.)
* TextMeshPro 패키지 (UI 텍스트 출력을 위해 필요합니다.)

### 2. 프로젝트 설정

1.  본 저장소를 클론하거나 다운로드합니다.
2.  Unity Hub에서 프로젝트를 엽니다.
3.  **씬 구성:**
    * 초기 진입 씬은 `MetaVerse` 또는 `Home` 씬으로 설정되어야 합니다.
    * `ScoreManager` 오브젝트에 `DontDestroyOnLoad`가 적용되어 있으므로, 초기 씬에 배치해야 합니다.
    * `UIManager` 오브젝트는 모든 UI 자식(`HomeUI`, `GameUI`, `ScoreUI`)을 가지고 상태에 따라 활성화/비활성화되어야 합니다.
    * 타워 쌓기 씬에는 `TheStack` 컴포넌트를 가진 오브젝트가 포함되어야 합니다.
